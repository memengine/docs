This guide covers the **Memory Passport** flow in MemoryOS.

Use this flow when:

- one person uses multiple AI agents
- those agents should not automatically share everything
- the user must explicitly approve what each agent can access

For the standard single-app pattern, use the tenant-scoped [Quickstart](/quickstart) instead.

## The model in one minute

Memory Passport adds three new concepts on top of tenant memory:

- a **universal user** with a one-time `uui_...` token
- a **global agent** with its own `agent_sk_...` API key
- a **permission grant** that limits access by category and access mode

Universal memory access always requires both credentials:

```http
Authorization: ApiKey agent_sk_...
X-MemoryOS-UUI: uui_...
```

## Supported categories

Current Memory Passport categories are:

- `preference`
- `fact`
- `goal`
- `procedure`
- `relationship`
- `expertise`

> `health` is **not** a valid category in the current backend and should not be requested.

## Step 1: Register a Memory Passport

The user creates a universal identity once.

```bash
curl -X POST https://api.memoryos.io/v1/uui/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "alice@example.com",
    "display_name": "Alice"
  }'
```

Example response:

```json
{
  "request_id": "f0abf8c5-2e65-4ef8-a2a0-4387d2b15182",
  "timestamp": "2026-04-21T08:45:13.233Z",
  "data": {
    "id": "e49fd5b1-3ebd-4e7b-a064-85db7a2e5c4a",
    "uui_token": "uui_4e12b0c3f5f4d3bd2c3dbf5f7f519ea2d07f3d0d447fa442",
    "email": "alice@example.com",
    "display_name": "Alice",
    "created_at": "2026-04-21T08:45:13.233Z",
    "is_active": true,
    "memory_count": 0
  }
}
```

The `uui_token` is shown **once** and should be stored by the user.

## Step 2: Show the agent profile before consent

Before asking for access, load the public profile for the agent:

```bash
curl https://api.memoryos.io/v1/agents/global/2beab5d0-741d-4ff8-9475-c29d9ea5a57e
```

Example response:

```json
{
  "request_id": "9482768d-c938-474d-bdc8-f54e1ec21be7",
  "timestamp": "2026-04-21T08:46:02.101Z",
  "data": {
    "id": "2beab5d0-741d-4ff8-9475-c29d9ea5a57e",
    "name": "Support Copilot",
    "description": "Customer support agent with access to approved user context.",
    "logo_url": null,
    "website_url": "https://example.com",
    "is_verified": false,
    "default_categories_requested": ["preference", "expertise"]
  }
}
```

Use this data in the consent UI so the user sees:

- who is requesting access
- whether the agent is verified
- which categories the agent is asking for

## Step 3: Create a permission grant

When the user approves access, create a grant with their `uui_...` token.

```bash
curl -X POST https://api.memoryos.io/v1/uui/me/grants \
  -H "Content-Type: application/json" \
  -H "X-MemoryOS-UUI: uui_4e12b0c3f5f4d3bd2c3dbf5f7f519ea2d07f3d0d447fa442" \
  -d '{
    "agent_id": "2beab5d0-741d-4ff8-9475-c29d9ea5a57e",
    "categories": ["preference", "expertise"],
    "access_type": "read_write",
    "duration_days": 90
  }'
```

Rules:

- `duration_days` may be `30`, `90`, `365`, or omitted for no expiry
- `access_type` may be `read_only` or `read_write`
- grant creation is idempotent per user plus agent pair

## Step 4: Add universal memories

Once the grant exists, the agent can write universal memories if the grant is `read_write`.

### Python SDK

```python
from memoryos.universal import UniversalMemory


with UniversalMemory(
    agent_api_key="agent_sk_...",
    uui_token="uui_...",
) as memory:
    result = memory.add(
        messages=[
            {
                "role": "user",
                "content": "Please remember that I prefer short, structured explanations and Python examples.",
            }
        ],
        metadata={"source": "consent-guide"},
    )

    print(result.status)
    print(result.quota_mode)
```

### REST API

```bash
curl -X POST https://api.memoryos.io/v1/universal/memories/add \
  -H "Content-Type: application/json" \
  -H "Authorization: ApiKey agent_sk_..." \
  -H "X-MemoryOS-UUI: uui_..." \
  -d '{
    "messages": [
      {
        "role": "user",
        "content": "Please remember that I prefer short, structured explanations and Python examples."
      }
    ],
    "metadata": {
      "source": "consent-guide"
    }
  }'
```

If the grant is `read_only`, MemoryOS returns:

- `403`
- `code: UAT_002`

## Step 5: Retrieve universal memories

The agent can then retrieve only the categories this user granted.

### Python SDK

```python
from memoryos.universal import UniversalMemory


with UniversalMemory(
    agent_api_key="agent_sk_...",
    uui_token="uui_...",
) as memory:
    result = memory.get(
        query="How should I tailor the response?",
        limit=5,
    )

    print(result.categories_available)
    print(result.permission_error)
    print(result.system_prompt_addition)
```

### REST API

```bash
curl -X POST https://api.memoryos.io/v1/universal/memories/retrieve \
  -H "Content-Type: application/json" \
  -H "Authorization: ApiKey agent_sk_..." \
  -H "X-MemoryOS-UUI: uui_..." \
  -d '{
    "query": "How should I tailor the response?",
    "limit": 5,
    "format": "bullets"
  }'
```

Response behavior:

- if a grant exists, MemoryOS searches the `universal_memories` collection and returns prompt-ready context
- if no grant exists, MemoryOS returns an empty list with:
  - `permission_error: "no_grant_for_user"`
- if the grant was revoked, the next retrieve becomes empty without exposing anything else about that user

## Step 6: Let the user manage or revoke access

The user can list active grants at any time:

```bash
curl https://api.memoryos.io/v1/uui/me/grants \
  -H "X-MemoryOS-UUI: uui_..."
```

Revoke one grant:

```bash
curl -X DELETE https://api.memoryos.io/v1/uui/me/grants/ef2143f3-8f7d-4a70-9f10-1c0c4c44f922 \
  -H "X-MemoryOS-UUI: uui_..."
```

Delete the entire Memory Passport and all universal memories:

```bash
curl -X DELETE https://api.memoryos.io/v1/uui/me \
  -H "X-MemoryOS-UUI: uui_..."
```

## TypeScript today

The dedicated universal client is available in the Python SDK today.

If you are building the consent or agent flow in TypeScript right now, call the REST endpoints directly and send:

```http
Authorization: ApiKey agent_sk_...
X-MemoryOS-UUI: uui_...
```

## Privacy guarantees

Memory Passport is designed so that:

- one agent cannot see which other agents were granted access
- a denied or revoked grant produces empty retrieve results rather than leaking metadata
- universal memories are stored separately from tenant-scoped `/v1/memories/*` data
